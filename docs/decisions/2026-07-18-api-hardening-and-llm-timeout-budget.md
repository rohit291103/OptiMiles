## Decisions

1. **Root cause of the 40â€“46s latency found and fixed: unbounded LLM calls, not a slow pipeline.** Instrumented reproduction showed OpenRouter's free tier returning 429 with `Retry-After: 29`, and the OpenAI client inside PydanticAI (default `max_retries=2`) honouring those sleeps â€” a single `model.complete()` burned ~44s then failed, after which the template fallback shipped anyway. Fix: `asyncio.wait_for` budgets at every LLM call site, mirroring `education.py`'s existing pattern. `extract_intent` gets a 10s bound; `narrate` gets a 10s **total** budget shared across its draft + regeneration via a deadline (reviewer catch: independent per-attempt bounds would still stack to 20s). Worst-case LLM time per request is now 10s (Stage 1) + 10s (Stage 10) = 20s, leaving ~10s of the 30s budget for the deterministic stages. Measured live: 45.2s â†’ 10.06s on the previously failing scenario. A timed-out attempt falls straight to the template â€” retrying a stalled provider would just burn a second budget.
2. **Product API versioned under `/v1`** (`main.py` router prefixes); `/health` deliberately stays at the root â€” liveness probes are infrastructure, not product surface. Frontend `api.ts` builds its base URL as `<host>/v1`; `NEXT_PUBLIC_API_BASE_URL` stays host-only.
3. **Missing goal-lineage indexes added (migration 0005).** `recommendation_outputs(goal_id, created_at DESC, id DESC)` exactly serves the saved-goals lateral latest-pick; `simulation_results(goal_id)` serves the delete path. Both were per-goal sequential scans â€” invisible at today's row counts, degrading with growth.
4. **`updated_at` made true (migration 0006).** Five tables carried `updated_at` that nothing ever set; a standard `BEFORE UPDATE` trigger (`public.set_updated_at()`) now stamps it in the database where no future write path can forget it. A column that lies is worse than no column.
5. **`POST /goals/recommendation/save` returns `saved_goal_id`.** `_persist` had discarded the ids `persist_recommendation` minted; the client had to re-fetch the list to find its own save. `persisted` semantics unchanged; the id ships only when the write actually landed. The wizard's "Saved âś“" now links straight to `/goals/{id}`.
6. **`GET /goals` is paginated** (`limit` 1â€“100 default 50, `offset` â‰Ą 0, 422 outside bounds) with a stable `created_at DESC, id DESC` order. The repository query is never unbounded again.
7. **Per-IP rate limiting on the anonymous compute-heavy endpoints** (`/v1/simulations` 10/min, `/v1/simulations/probe` 30/min): a hand-rolled fixed-window limiter (`api/ratelimit.py`) with an injected clock (deterministically tested, no sleeps) rather than `slowapi` â€” one sync process by design (build rule 7), so an external store would be infrastructure ahead of need. Expired windows are pruned past a threshold (reviewer catch: the key space is client-controlled, so the abuse brake must not itself grow unboundedly). 429 + `Retry-After`. Verified live with 12 concurrent requests: exactly 10Ă—200 + 2Ă—429.
8. **Minimal observability layer** (`api/observability.py`): one INFO line per request (method, path, status, duration_ms, request_id) + `X-Request-ID` response header (caller-supplied id echoed for cross-hop correlation). `logging.basicConfig` added in `create_app` â€” live verification caught that uvicorn's `--log-level` configures only its own loggers, so app INFO lines (including the narration-fallback warnings) were invisible in production. Deliberately not metrics/OTel.
