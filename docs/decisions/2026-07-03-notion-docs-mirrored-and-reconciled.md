# Decision Log — Notion Docs Mirrored into /docs + Blueprint Reconciled (v1.1)

**Date:** 2026-07-03
**Area:** architecture / product / docs

## Context

The Notion workspace ("Rohit -CS's Notion" → OptiMiles) held 17 documents the repo didn't have — including MVP Scope v2, the Constitution, a cluster of engine designs (Recommendation Engine Design/Workflow, RIF, Strategy Generation, Strategy Ranking, Reward Knowledge Engine Spec, Core Domain Model), five user/market research docs, and the Goal-to-Strategy UX journey. Per "Chats are temporary. Documentation is permanent," and ahead of starting backend implementation, the user asked to: update the repo scope doc, cross-check the new execution blueprint (`system-execution-flow-v1.md`, written 2026-07-03) against the Notion engine docs, and mirror everything into `/docs` properly organized.

## Decisions

1. **MVP Scope v2 is now canonical** — `docs/prd/mvp-scope-v2.md` (mirrored + reconciled); `mvp_scope_1.md` carries a superseded banner and is retained as history. Material scope changes vs v1: airlines narrowed to **KrisFlyer + Air India Maharaja Club** (Emirates/Qatar/Vistara out; Vistara merged into Air India, "Flying Returns" renamed); **ICICI Rewards dropped**; primary-user definition added (₹50k–₹2L/month); "Maximize Existing Cards First" elevated to scope principle; six core capabilities incl. Existing Portfolio Optimization; **performance target tightened to 30 seconds**; config-over-code extensibility as an NFR. The 8-card universe and Singapore/London/New York routes carry over unchanged.
2. **The execution blueprint survived the cross-check and was bumped to v1.1** with adoptions, not restructuring. The Notion workflow's 10 phases map cleanly onto the blueprint's 11 stages (mapping table in `recommendation-engine-design-v1.md` §6.1). Adopted into v1.1: preference-aware ranking weights (DP-07) + portfolio-utilization ranking dimension; explicit strategy-validation (Stage 7 exit) and pruning/dominance-removal (Stage 9 pre-pass) steps; `PortfolioAssessment` as a named feasibility-gate output; the Recommendation Package as the `FinalRecommendation` payload contract; 30s budget (streaming = planned API shape, not later optimization).
3. **Two conflicts resolved against the Notion drafts** (recorded in `recommendation-engine-design-v1.md` §6.2): (C-1) RED-001's "Strategy Generation ⚠️ AI-assisted" marker contradicts its own DP-05 and the Constitution's "Rules Before AI" — resolved to **fully deterministic** generation; (C-2) "generate every feasible strategy" (DP-04/BR-03) is combinatorially unbounded — the blueprint's **bounded archetype generation** stands as the MVP realization of "generate broad, recommend narrow." Also confirmed: no standalone Opportunity/Ranking/Strategy engines (they're modules — consistent with the 2026-07-03 blueprint decision), and the Notion workflow's missing explicit Simulation stage is an improvement the Notion side should adopt from the blueprint.
4. **Engine docs consolidated, not mirrored 1:1.** Three overlapping recommendation docs (RED-001 + REW-001 + RIF-001) → one `recommendation-engine-design-v1.md`; two module designs (SGE-001 + SRE-001) → one `optimization-engine-spec-v1.md` framed as Optimization Engine module specs with all business rules (BR tables) preserved. Rationale: seven overlapping engine docs would guarantee drift; the repo keeps one doc per system.
5. **RKE spec mirrored with a schema-gap analysis** (`reward-knowledge-engine-spec-v1.md` §12): the highest-priority delta vs `db-schema-v1.md` is **Reward Currency as a first-class entity** (AD-04 — transfer relationships belong to currencies, not cards; HDFC's three cards share one currency and per-card transfer rows duplicate/drift). Recommended for db-schema v1.1 **before** Knowledge Engine scaffolding. Merchants/Promotions (AD-06, needed for SmartBuy/portal opportunities) deliberately deferred post-MVP; rule versioning stays post-MVP with snapshot-version lineage as the MVP audit trail.
6. **Routing of the rest:** Constitution → `docs/prd/optimiles-constitution-v1.md`; Core Domain Model → `docs/architecture/core-domain-model-v1.md` (with added pipeline/DB mapping table); five user/market docs → `docs/research/` (`user-personas-v1`, `user-pain-points-v1`, `jobs-to-be-done-v1`, `current-customer-journey-v1`, `value-propositions-v1`); Goal-to-Strategy journey → `docs/ux/goal-to-strategy-user-journey-v1.md` (with pipeline-source column added to the dashboard sections); AI Collaboration Protocol → `docs/prompts/ai-collaboration-protocol-v1.md`. Notion's `postgres_schema_singapore_rewards_engine_v1` is **byte-identical** to the repo's `db-schema-v1.md` — not duplicated. Notion's MVP Scope v1 and the older "Reward Knowledge Engine Design" are superseded by their v2/Specification versions — not mirrored.
7. **All mirrors carry a provenance header** (source Notion page + mirror date) and were normalized to repo naming (`kebab-case-topic-v1.md`).
